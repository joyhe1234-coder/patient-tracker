# Implementation Tasks: Insurance Group

## Task Overview

This implementation adds an `insuranceGroup` field to the Patient model that associates each patient with the healthcare system from which they were imported. The feature extends the import pipeline to persist the selected `systemId`, adds server-side filtering to the data API, and adds a dropdown filter in the UI. The feature touches four layers: database schema, import pipeline, data API, and frontend UI.

**Key Components:**
- 1 database migration (schema + data migration)
- 8 modified backend files (schema, import executor, data handlers, types)
- 3 modified frontend files (StatusFilterBar, MainPage, PatientGrid)
- 0 new files
- 4 test frameworks (Jest backend, Vitest frontend, Cypress E2E, MCP Playwright visual)

**Dependency Strategy:**
Tasks are ordered to minimize blocking:
1. Database schema and migration (enables all other work)
2. Backend implementation (import pipeline + data API)
3. Frontend implementation (UI components)
4. Backend unit tests (Jest)
5. Frontend component tests (Vitest)
6. E2E tests (Cypress)
7. Visual review (MCP Playwright)

## Steering Document Compliance

**structure.md conventions followed:**
- Database changes in `backend/prisma/schema.prisma` (Prisma migration workflow)
- Backend service changes in `backend/src/services/import/` (existing modules)
- Backend route handler changes in `backend/src/routes/handlers/` (existing modules)
- Frontend UI changes in `frontend/src/components/layout/` and `frontend/src/pages/` (existing modules)
